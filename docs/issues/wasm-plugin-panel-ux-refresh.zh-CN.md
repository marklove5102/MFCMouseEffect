# WASM 插件面板可读性与审美重构（2026-02）

## 背景
- 旧版面板把策略、目录、控制、诊断全部平铺在一个网格里。
- 关键信息（是否加载、当前插件、运行后端）缺少首屏层级。
- `清单路径` 与 `配置清单路径` 在多数情况下重复展示，容易让用户误解为异常。

## 本次调整
1. 信息架构重排
- 新增顶部摘要条：`插件加载状态 / 运行时后端 / 当前插件 / 最近调用指标`。
- 新增三块分区：
  - `插件目录`：扫描、选择、加载、目录错误与扫描根路径。
  - `运行策略`：启停控制、回退策略、预算参数与运行预算显示。
  - `运行诊断`：渲染统计、预算标记、预算原因、解析错误与宿主错误。

2. 路径表达优化
- 统一主展示为 `清单路径`（优先运行时路径，其次配置路径）。
- 当运行时路径与配置路径一致时，仅显示一条并附加“已与配置路径同步”徽标。
- 仅在两者不一致时额外展示 `配置清单路径`，降低冗余。

3. 视觉层级优化
- 新增分区容器、标题、副说明和摘要卡片样式。
- 新增内联状态徽标（同步状态）。
- 保持现有配色体系，不引入新的主题系统。

4. 运行策略首行优化（后续微调）
- 将 `运行控制按钮（启用/禁用/重载）` 上移到“运行策略”分区第一行，符合操作优先路径。
- 将标签从“运行时控制”调整为“即时操作”，更贴近该行的行为语义（立即生效动作，而非只读状态）。

## 影响文件
- `MFCMouseEffect/WebUIWorkspace/src/wasm/WasmPluginFields.svelte`
- `MFCMouseEffect/WebUI/styles.css`
- `MFCMouseEffect/WebUI/i18n.js`

## 回归关注点
- 语言切换时，新文案键是否正确替换。
- 小屏下摘要条是否退化为单列，避免挤压。
- `配置清单路径` 在“相同/不同”两种场景下展示是否符合预期。

## 后续修正（i18n 动态渲染）
- 现象：中文环境下，“操作结果”行在操作后动态出现，标签显示英文 `Operation result`。
- 根因：该行是条件渲染节点，使用静态 fallback 文本，未稳定经过动态 i18n 文本绑定。
- 修正：改为组件内直接调用 `text('label_wasm_operation_result', ...)` 渲染，确保节点后插入时也按当前语言显示。
