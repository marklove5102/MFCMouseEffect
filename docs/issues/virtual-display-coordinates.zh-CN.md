# 虚拟副屏/平板副屏：鼠标坐标偏移（定位与兜底）

## 状态
- 已修复（2026-01）：默认启用坐标归一化，虚拟/平板副屏上的偏移基本消除；如仍有问题，请记录驱动/软件名称和 DPI 设置反馈。

## 现象
- 平板作为副屏（通过虚拟显示器/投屏软件模拟）时：
  - 点击/滚轮/拖尾/悬停特效出现明显偏移
  - 特效被画到主屏、或看似“消失”（其实画到屏幕范围外）
- 在真实外接显示器上不一定复现（取决于各显示器 DPI/缩放组合）

## 可能原因（宏观）
- **虚拟显示器驱动/副屏软件**在坐标映射或 DPI 缩放处理上与系统/普通显示器不一致。
- **坐标系混用**：
  - `WH_MOUSE_LL` 给出的 `MSLLHOOKSTRUCT.pt` 与 `GetCursorPos` 在某些场景下可能不在同一坐标空间。
  - 多显示器 + 不同缩放比例（Per-monitor DPI）时更容易触发。

## 我们的处理策略（兜底）
- 对输入事件点做一致性校验：
  - 如果 `hook` 的 `pt` 与 `GetCursorPos` 的点偏差超过阈值（目前 64px），则优先使用 `GetCursorPos`。
- 这样做的目标是：
  - 优先保证“特效跟着真实鼠标走”，即使虚拟显示器坐标偶尔异常。
  - 对正常显示器/正常坐标不产生影响（小偏差会继续使用 hook 点）。

## 影响范围
- 该兜底会影响：移动（Trail/Hover）、点击、滚轮、按下（Hold）。
- 这是 best-effort 修复：如果虚拟显示器驱动返回的光标坐标本身就异常，仍可能出现问题。

## 排查建议
- 对比同一时刻：
  - `MSLLHOOKSTRUCT.pt`
  - `GetCursorPos`
- 如果只在虚拟副屏上差异巨大，优先怀疑副屏软件/驱动；真实外接屏也同样错，则需要进一步检查 DPI/坐标处理。
