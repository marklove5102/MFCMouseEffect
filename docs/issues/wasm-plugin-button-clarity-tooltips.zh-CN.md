# WASM 面板按钮语义与悬浮提示优化

## 背景
- “保存扫描路径”和“保存策略”两个按钮语义不够直观，容易误解为同一类保存行为。
- WASM 面板按钮较多，缺少悬浮提示时，用户难以快速判断按钮副作用。

## 调整内容
- 按钮文案优化：
  - `保存扫描路径` -> `应用扫描路径`
  - `保存策略` -> `保存回退与预算`
- 运行时控制区改造：
  - 原“启用/禁用”双按钮改为单一开关按钮。
  - 开关文案直接显示状态：`正在使用插件` / `未使用插件`，便于一眼判断当前是否走插件路径。
- 为 WASM 面板全部核心按钮补充 `title` 悬浮提示（并接入 i18n）：
  - 应用扫描路径、刷新插件列表、加载所选插件、导入插件目录、导出全部插件
  - 运行开关、重载插件、保存回退与预算、恢复默认策略

## 交互语义说明
- `应用扫描路径`：保存 `catalog_root_path` 到配置，并立即按该路径重新扫描插件列表。
- `保存回退与预算`：保存 `fallback_to_builtin_click` 与预算参数（输出缓冲、最大命令、最大执行时长）到 `config.json`。
